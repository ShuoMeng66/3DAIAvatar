# ElderTalk 前端设计文档

## 设计理念

### 核心原则
1. **极简交互**：老人不应面对复杂操作，核心操作不超过 3 个按钮
2. **视觉优先**：大字体、大按钮、高对比度，确保可读性
3. **语音为主**：默认语音交互，文字输入为辅助
4. **温暖亲切**：使用暖色系和圆润字体，营造陪伴感
5. **无干扰**：无弹窗、无广告、无复杂菜单

### 老年用户特点
- 视力下降：需要 ≥20px 字号，高对比度
- 手指不灵活：需要 ≥64px 触摸目标
- 对技术陌生：操作逻辑要符合直觉（按住=说话，松开=发送）
- 容易孤独：需要温暖、亲切、有耐心的交互体验
- 短时记忆弱：不需要记住复杂操作流程

## 配色方案

### 主色调
| 颜色 | 色值 | 用途 |
|------|------|------|
| 米色背景 | #FFF8F0 | 页面主背景 |
| 深米色 | #F5E6D3 | 渐变底部 / 卡片背景 |
| 暖橙色 | #F59E0B | 主按钮、强调色 |
| 深橙色 | #D97706 | 按钮悬停 |
| 深棕色 | #3D2C0A | 主文字色 |
| 浅棕色 | #6B5A3E | 辅助文字色 |
| 边框色 | #E5E0D8 | 输入框边框 |

### 对比度
- 文字/背景对比度 ≥ 7:1（AAA 级）
- 按钮文字/背景对比度 ≥ 4.5:1（AA 级）

## 字体

### 首选字体
- **思源黑体 (Noto Sans SC)**：Google Fonts CDN 引入
- 字重：400（正文）、500（标题）、700（强调）、900（超大标题）
- 特点：圆润、现代、中文显示优秀

### 字号规范
| 元素 | 字号 | 字重 |
|------|------|------|
| 页面标题 | 36px | 900 |
| 字幕文字 | 28px | 700 |
| 按钮文字 | 22px | 700 |
| 正文 | 20px | 400 |
| 辅助文字 | 16px | 400 |

## 布局规范

### 触控目标
- 最小触摸区域：64×64px
- 按钮间距：≥16px
- 输入框高度：≥64px

### 页面结构
- 顶部：标题栏（高度 64px）
- 中间：内容区（flex-1，自适应）
- 底部：操作栏（高度 128px，固定）

### 动画
- 不使用花哨动画
- 仅使用简单的过渡效果（transition: 0.2s）
- 按钮按下有轻微缩放反馈（scale: 0.98）

## 图标

使用 Lucide React 图标库：
- `Mic`：麦克风（录音中）
- `MicOff`：麦克风关闭
- `StopCircle`：停止
- `Send`：发送
- `Settings`：设置
- `Monitor`：全息屏
- `MessageCircle`：聊天

## 设计参考

### 老年友好 UI 参考
- Apple 辅助功能设计指南
- Google Material Design Accessibility
- WCAG 2.1 AA/AAA 标准
- 「老年人模式」App 设计：大字体、大按钮、简化流程

### 同类产品参考
- 小度智能屏（百度）：语音交互为主的老人模式
- 天猫精灵 CC：大屏语音交互
- 京东叮咚：老人陪伴模式

### 配色参考
- 温暖、亲切的色调，避免冷色系
- 参考日本「和」设计风格：米色、暖橙、木色
- 避免纯白（刺眼），使用微暖的米色
